# Part 1. AI 프레임의 전환 : 도구에서 대리인으로

## 1.1 AI 에이전트의 정의

**페이지 구성:** 3단 동등 나열 (좌→우 또는 상→하)

각 블록 구조: 정의 레이블 + 한 줄 설명 + 부제 + 키워드 3개

### 블록 1 — 관계적 정의
- **레이블:** 관계적 정의
- **한 줄 설명:** 사용자와 AI의 관계가 '명령'에서 '협업'으로 바뀌는 소통 방식
- **부제:** 외적 현상
- **키워드:** 의도기반, 위임, 신뢰

### 블록 2 — 구조적 정의
- **레이블:** 구조적 정의
- **한 줄 설명:** LLM의 추론과 도구가 결합되어 스스로 움직이는 지능적 아키텍처
- **부제:** 내부적 원리
- **키워드:** 추론, 도구 활용, 계획

### 블록 3 — 가치적 정의
- **레이블:** 가치적 정의
- **한 줄 설명:** 실제 세계의 과업을 종결짓는 생산 주체
- **부제:** 목적적 본질
- **키워드:** 실행, 완결, 가치창출

## 1.2 디자인 R&R의 확장

**페이지 구성:** 상단 2블록 + 하단 AS-IS→TO-BE 테이블

**도입부 카피:** "에이전트를 설계할 것인가? 에이전트로 디자인할 것인가?"

### 상단 블록

**블록 1 — Designing FOR Agents: 지능의 설계자**
디자이너가 설계하는 대상이 '정적인 인터페이스'에서 '동적인 지능'으로 확장.
유저가 마주할 에이전트의 사고방식과 행동양식을 결정.

**블록 2 — Designing WITH Agents: 워크플로우 혁신가**
디자이너가 일하는 방식 자체가 변화.
에이전트를 단순한 도구가 아닌 '지능형 파트너'로 부리며 생산성을 극대화.

### 하단 테이블 (AS-IS → TO-BE)
컬러 구분: a,b행 = FOR Agents 계열 / c,d,e행 = WITH Agents 계열

| 구분 | AS-IS | TO-BE |
|------|-------|-------|
| a. 설계 대상 | 애플리케이션 화면 | 에이전트의 사고와 행동 |
| b. 핵심 역량 | 비주얼 레이아웃, 인터랙션 | 맥락 이해, 추론 설계, 가치 필터링 |
| c. 일하는 방식 | 툴을 활용한 수동 제작 | 에이전트와 협업하는 자동화 루틴 |
| d. 산출물 | 디자인 가이드, 프로토타입 | 판단 정책, 프롬프트, 작동하는 PoC |
| e. 협업 파트너 | 기획자, 개발자 | 개발자 + 에이전트 + 데이터 |

## 1.3 디자인 대상의 전환, 앱에서 에이전트의 시대로

**페이지 구성:** 테이블 (앱 / 에이전트 2컬럼, 3행 + 각 행 하단 인사이트 병합셀)

**도입 카피:** "사용자가 기능을 찾아다니는 시대에서 에이전트가 목적을 대신 수행하는 시대로"

### 테이블

| 구분 | 앱 | 에이전트 |
|------|-----|---------|
| **1. UI to Intent** 메뉴 중심에서 의도 중심으로 | 사용자가 메뉴를 탐색하고 기능을 하나씩 실행하는 도구함 (유저가 길을 알아야 함) | 사용자의 추상적인 의도를 이해하고 최적의 경로를 스스로 생성 (에이전트가 길을 찾음) |
| **인사이트** (병합셀) | How to use의 정교함보다 What to do의 정확성이 핵심 | ← |
| **2. Feature to Goal** 기능의 나열에서 과업의 완결로 | '송금하기', '장바구니 담기' 같은 파편화된 기능을 제공. 최종 결과는 유저의 노동으로 완성 | '이번 주말 여행 준비해 줘'라는 하나의 목표를 이해, 여러 기능을 엮어 과업을 완결 |
| **인사이트** (병합셀) | 성공지표가 '체류시간(Retention)'에서 '과업 성공률(Completion Rate)'로 이동 | ← |
| **3. Passive to Proactive** 수동적 반응에서 선제적 대응으로 | 사용자가 아이콘을 눌러야만 움직이는 수동적 존재 | 상황과 맥락을 모니터링하다가 필요할 때 먼저 말을 걸거나 액션 |
| **인사이트** (병합셀) | 인터랙션 트리거가 유저의 손가락이 아닌 에이전트의 상황 판단 | ← |

## 1.4 디자인 프로세스의 진화, 제작에서 지휘로

**페이지 구성:** 테이블 (UX시대 / AX시대 2컬럼, 4행 + 각 행 하단 인사이트 병합셀)
텍스트 밀도가 높은 경우 맥락 유지하며 압축.

**도입 카피:** "에이전트를 파트너로 부리며 일할 때 발생하는 태도와 관점의 변화"

### 테이블

| 구분 | UX시대 | AX시대 |
|------|--------|--------|
| **1. 리서치** Synthesis over Collection | 유저 인터뷰, 데스크 리서치로 파편화된 정보를 직접 수집하는 데 많은 시간 할애 | 에이전트가 방대한 데이터를 실시간 요약·대조. 디자이너는 새로운 인사이트를 '합성' |
| **인사이트** (병합셀) | 정보 탐색의 물리적 시간이 줄어들고 질문의 질이 리서치의 성패를 결정 | ← |
| **2. 아이데이션** Co-creation over Ideation | 화이트보드 앞에서 포스트잇을 붙이며 스스로 아이디어를 짜냄 | 코딩/이미지/텍스트 에이전트와 대화하며 수많은 시안을 초단위로 검토 |
| **인사이트** (병합셀) | 무에서 유를 만드는 고통 대신, 제안받은 수많은 가능성 사이에서 최적의 논리를 선택하고 정교화 | ← |
| **3. 프로토타이핑** Implementation over Description | 실제처럼 보이는 가짜 화면을 그려 개발자에게 설명 (Draw to show) | 코딩 에이전트와 함께 실제로 돌아가는 지능의 로직을 즉시 코드로 구현 (Build to test) |
| **인사이트** (병합셀) | 시각적 묘사에 머물던 프로토타이핑이 작동하는 로직 검증으로 격상. 그림이 아닌 '작동하는 실체'로 소통 | ← |
| **4. 검증** Evaluation over Observation | 사용자가 버튼을 잘 누르는지 뒤에서 지켜보는 관찰자 | 수만 개의 가상 시나리오를 에이전트에 통과시켜 답변 품질을 점수화, 로직을 수정하는 시스템 평가자 |
| **인사이트** (병합셀) | 한두 명의 유저 반응을 보는 것을 넘어 시뮬레이션을 통해 테스트하고 튜닝 | ← |
