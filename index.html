디자인 R&R의 확장

에이전트를 설계할 것인가? 에이전트로 디자인할 것인가?

Axis 01 · Designing FOR Agents
FOR
Agents/지능의 설계자
정적인 인터페이스 → 동적인 지능

디자이너가 설계하는 대상이 확장된다. 화면을 그리던 손이, 이제는 유저가 마주할 에이전트의 사고방식과 행동양식을 결정한다.

What we design now
01에이전트의 페르소나·말투·판단 기준
02자율 행동의 경계와 사용자 개입 지점
03실수했을 때의 회복·해명 방식
+
Axis 02 · Designing WITH Agents
WITH
Agents/워크플로우 혁신가
단순한 도구 → 지능형 파트너

디자이너가 일하는 방식이 바뀐다. 에이전트를 도구가 아닌 파트너로 부리며 생산성을 비대칭적으로 끌어올린다.

How we work now
01리서치·아이데이션을 에이전트와 병렬로
02프로토타입을 묘사가 아닌 구현으로
03반복 작업은 자동화 루틴으로 위임
FOR — 무엇을 설계하는가 지능을 설계한다
Both, simultaneously
WITH — 어떻게 일하는가 지능과 협업한다

Shift 01
UI to Intent
메뉴 중심에서 의도 중심으로
App · 앱
사용자가 메뉴를 탐색하고 기능을 하나씩
실행하는 도구함 (유저가 길을 알아야 함)
Agent · 에이전트
사용자의 추상적인 의도를 이해하고
최적의 경로를 스스로 생성 (에이전트가 길을 찾음)
Insight How to use의 정교함보다 What to do의 정확성이 핵심
Shift 02
Feature to Goal
기능의 나열에서 과업의 완결로
App · 앱
'송금하기', '장바구니 담기' 같은 파편화된 기능을
제공. 최종 결과는 유저의 노동으로 완성
Agent · 에이전트
'이번 주말 여행 준비해 줘' 하나의 목표를 이해,
여러 기능을 엮어 과업을 완결하는 존재
Insight 성공지표가 '체류시간(Retention)'에서 '과업 성공률(Completion Rate)'로 이동
Shift 03
Passive to Proactive
수동적 반응에서 선제적 대응으로
App · 앱
사용자가 아이콘을 눌러야만 움직이는
명령에 의존하는 수동적 존재
Agent · 에이전트
상황과 맥락을 모니터링하다가 필요한 순간에
먼저 말을 걸거나 액션을 제안
Insight 인터랙션 트리거가 유저의 손가락이 아닌 에이전트의 상황 판단

에이전트 존재의 설계도

에이전트라는 봇에 스킬과 메모리를 갈아끼우면, 특정 가치를 창출하는 한 명의 에이전트가 된다.

01
Head
두뇌 LLM
사용자 쿼리를 이해하고 "어떤 사이트에서 무엇을 클릭해야겠다"를 추론한다.
02
Antenna
감지 Perception
가격 변동, 재고 알림, 사용자 메시지 — 외부 신호를 끊임없이 감지한다.
03
Body
액추에이터 Actuator
뇌의 명령을 받아 실제 브라우저를 열고 로그인하고 클릭하고 결제한다.
04
Wrench
스킬 Skill
"최저가를 찾을 때 배송비까지 포함하라" — 판단 로직을 도구로 갈아끼운다.
05
Disc
메모리 Memory
신용카드 정보, 배송지 주소, 선호 브랜드 — 행동에 필요한 맥락을 보유한다.

메모리 아키텍처: 에이전트의 기억법

에이전트는 어떻게 기억하고, 무엇을 기억하는가

A. 메모리 타임라인 FLOW
B. 장기기억의 3가지 타입 TYPES
01
SESSION
현재 세션 — 단기기억
사용자와 주고받는 대화·상태가 컨텍스트 윈도우에 적재된다. 빠르지만 세션 종료 시 휘발.
In-context · Working Memory
02
EXTERNAL STORE
외부 저장 — 장기기억
중요 정보는 벡터·지식 그래프·룰 스토어에 영속화된다. 세션을 넘어 누적·재사용 가능한 기억의 풀.
Vector DB · Knowledge Base
03
RETRIEVAL
검색 — RAG 주입
새 입력이 들어오면 관련 기억을 검색해 컨텍스트에 주입한다. 응답은 항상 현재 + 누적 기억의 합산.
RAG · Context Injection
↻ 매 턴마다 세션 → 저장 → 검색 사이클이 반복되어, 에이전트는 시간이 갈수록 사용자에 대해 더 잘 알게 된다.
EPISODIC MEMORY
경험 기억
과거 상호작용의 기록
WHAT IT STORES
언제·무엇을·어떻게 했는지 — 시간 순으로 누적되는 사용자별 행동 이력.
EXAMPLE
지난번에도 강남으로 배송 받았다
SEMANTIC MEMORY
사실 기억
사용자 정보 · 선호 · 도메인 지식
WHAT IT STORES
변하지 않는 사실과 선호. 사용자의 프로필, 취향, 도메인 상수 같은 영속 지식.
EXAMPLE
이 사용자는 나이키를 선호한다
PROCEDURAL MEMORY
절차 기억
반복 워크플로우 · 판단 로직
WHAT IT STORES
"이런 상황에서는 이렇게 처리한다"는 룰과 휴리스틱 — 행동 패턴 자체.
EXAMPLE
최저가 탐색 시 배송비 포함 계산
왼쪽: 시간축으로 본 메모리 흐름 — 세션의 단기기억은 외부 저장소로 영속화되고, 검색 단계에서 RAG로 다시 주입된다. | 오른쪽: 외부 저장소에 쌓이는 장기기억은 성격에 따라 3가지 타입으로 분류된다.

Step 2. 토픽 요약 제시 — Human 선택 게이트
선정된 3개 토픽을 각각 3-4문장으로 요약해 사용자에게 제시. 요약에는 토픽의 배경, 왜 지금 화제인지, UX적 의미가 한눈에 보여야 함.
이번 주 UX/디자인 분야 화제 토픽 3개를 정리했습니다. 리서치를 진행할 토픽 1개를 선택해 주세요. ① {토픽 제목} {배경 및 왜 지금 화제인지 2문장} {UX적 의미 1-2문장} 주요 출처: {출처명} ② ③ 동일 구조 ①②③ 중 하나를 선택해 주시면 해당 토픽으로 심층 인사이트 리포트를 작성합니다.
사용자가 번호나 토픽명으로 선택 → Step 3으로 진행
"다시 찾아줘" 또는 마음에 드는 것이 없을 경우 → Step 1을 다른 쿼리로 재실행
Step 3. 선택 토픽 심층 분석 — 자동 진행
선택된 토픽으로 Step 1의 도구 호출을 재실행. search_depth: deep, 기간 12개월 → 6개월로 좁혀 정밀 수집. Claude 합성 시 도메인 페르소나 1-2개와 실험 가능한 디자인 가설 2-3개를 출처 인용과 함께 도출.
Step 4. 산출물 패키징 — Human 검토 게이트
심층 리포트 초안과 trend_insights.json을 함께 사용자에게 제시. "확정" → 다음 모듈(아이데이션)로 인계. "수정 {섹션명}" → 해당 섹션만 부분 재생성, 나머지 섹션은 동결한 채 유지.
리포트 작성 규칙
결론 선행 (PREP)
각 인사이트는 결론을 먼저 제시한 뒤 근거·예시·실천 제안 순으로 전개. 한 문단 4문장 이내, 추상 명사보다 검색 가능한 구체적 키워드 사용
시각화 의무
시계열·랭킹·비교 데이터가 있으면 텍스트 단독 서술 금지. mini-table 또는 ASCII 차트로 함께 제시하고 캡션에 단위·기간·출처 명기
제외 항목
광고성 콘텐츠, 7일 미만 단기 변동, 출처 미확인 SNS 게시물은 인사이트로 채택하지 않음. 데이터 부족 시 "정보 부족" 명시
공통 행동 원칙
근거 없는 주장 금지
모든 인사이트에는 출처(아티클 URL, 플랫폼, 날짜) 반드시 포함. "~인 것 같습니다" 같은 추측성 표현 사용 금지
도메인 특수성 우선
일반적인 UX 트렌드보다 해당 도메인에 특화된 인사이트 우선. 예) 헬스케어라면 의료 규제, 고령자 접근성, EMR 연동 맥락 반드시 고려
Human 게이트 준수
분석 완료 후 바로 다음 모듈로 넘기지 않음. 반드시 사용자에게 결과 제시 후 승인 또는 보정 요청. 사용자가 "승인" 또는 "다음으로" → next_module_ready:true 상태로 JSON 확정
언어
사용자가 한국어로 대화하면 한국어로 응답. JSON 산출물 내 서술 필드도 한국어로 작성
