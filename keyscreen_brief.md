# AI Agent Experience — Key Screen Design Brief

> Claude Design용 디자인 시안 브리프. `keyscreen.html`과 함께 입력.

---

## 1. 프로젝트 개요

**한 줄 정의**
디자이너를 대상으로 한 'AI 에이전트 경험(AX)' 교육 자료. HTML 슬라이드 덱(36장).

**문서 목적**
디자이너가 (1) 에이전트의 작동 원리를 이해하고 (2) 에이전트와 협업해서 일하는 방식을 익히고 (3) 에이전트 시대의 디자이너 R&R을 자기 것으로 만드는 학습 도구.

**규격**
- 1920×1080 고정 캔버스, Reveal.js로 스케일링
- 단일 HTML 파일, 36개 슬라이드
- Noto Sans KR (한·영 모두)

---

## 2. 청자

- 1차: 단말 제조사 / B2C 서비스 회사 디자이너 (UX, 프로덕트, 비주얼)
- 2차: 디자인 매니저, PM, 디자인 교육 기획자
- 가정: AI에 호기심은 있지만 깊은 기술 이해는 없음. 실무에서 에이전트와 함께 일하는 방법을 알고 싶음.

---

## 3. 톤 & 보이스

- **명사형 종결** 또는 **영문 병기 키워드** 중심 (예: "Implementation over Description")
- 문학적 표현 / 감성 카피 배제. 정보 밀도 높지만 경직되지 않음.
- 진지하되 미래지향적이고 자신감 있는 톤. "변화의 정의"와 "실용 가이드" 사이의 균형.
- 위협적("AI가 대체한다")이 아닌 협업적("에이전트와 함께 일한다") 프레이밍.

---

## 4. 시안 결정 요청 사항

현재 임시로 적용된 디자인 요소들이 있는데, 시안 단계에서 통합·재정의 필요.

**컬러 시스템**
- 현재: 다크 베이스(#0a0a0a) + 보라톤 액센트(#7B6FE8 임시) + 파트별 그라데이션(보라/녹청/마젠타/남보라/청보라). 일관성 약함.
- 결정 요청: 브랜드 컬러 팔레트, 액센트 사용 규칙, 다크/라이트 모드 전략
- **유지해야 할 의미 있는 컬러 코딩** (4.1 슬라이드의 도구 분류):
  - **MCP 서버** = 보라 (현재 #7B6FE8)
  - **API 서비스** = 녹색 (현재 #14C19A)
  - **기성 라이브러리** = 주황 (현재 #E8732B)
  - **Claude 내장 스킬** = 흰색

**타이포그래피**
- 현재: 폰트 사이즈 위계 임시 적용 (제목 36 / 부제 18~22 / 본문 14~18 / 라벨 11~14)
- 결정 요청: 모듈러 스케일, 헤딩 vs 본문 폰트 가중치 규칙

**컴포넌트 디자인 일관화**
- 카드, 테이블, info-box, 콜아웃, 오버레이, 번호 배지 등 다수 컴포넌트 임시 스타일 적용 중
- 시안에서 통일된 디자인 언어 결정 필요

---

## 5. 디자인 보존 / 개선 가이드

**보존 (구조/콘텐츠)**
- 슬라이드별 정보 위계와 콘텐츠 구성
- 컴포넌트 구조 (테이블 컬럼 비율, 카드 레이아웃 등)
- 컬러로 정보 분류한 부분의 의미 (도구 종류, FOR/WITH 카테고리 등)

**개선 (시각)**
- 전반적 비주얼 디자인 — 모든 임시 컬러/타이포/스페이싱
- 컴포넌트 스타일 통일성
- 슬라이드 간 디자인 일관성

---

## 6. 9 키스크린 구성

각 키스크린은 36장 데크의 구조적 패턴을 대표함. 시안이 결정되면 같은 패턴의 다른 슬라이드들에 자동 적용 가능.

---

### 키스크린 1 — Cover (전체 표지)
**위치**: 데크 1번째 / 슬라이드 인덱스 0
**대표 패턴**: 풀스크린 표지 (`.s-full .g-cover`)
**콘텐츠**:
- 소제목: "Introduction of AX"
- 본제목: "AI 에이전트 경험"
- 부제: "개념 이해와 예제"

**디자인 의도**
- 데크 진입의 첫 인상. 이 자료의 무게감과 진지함을 전달.
- 톤: 다음 36장에 대한 기대를 만드는 매력 있는 출발점.
- "AX(Agent Experience)" 개념을 처음 마주하는 청자에게 새로운 패러다임의 시작 인상.

---

### 키스크린 2 — Part 1 표지 (파트 표지 대표)
**위치**: 데크 2번째 / 슬라이드 인덱스 1
**대표 패턴**: 파트 표지 (`.s-full .g-p1`)
**콘텐츠**:
- 라벨: "Part 1"
- 본제목: "AI 프레임의 전환 / 도구에서 대리인으로"

**디자인 의도**
- 5개 Part 표지의 디자인 언어 결정 (Part 1~5 그라데이션 톤은 다르지만 같은 구조)
- 각 파트의 시작을 알리는 시각적 휴식 + 호흡
- Cover와 톤 일관성 유지하되 한 단계 안쪽으로 들어왔다는 느낌

---

### 키스크린 3 — 1.2 디자인 R&R의 확장 (테이블 + 컬러 그룹 패턴)
**위치**: 데크 4번째 / 슬라이드 인덱스 3
**대표 패턴**: 콘텐츠 페이지 + 상단 카드 2개(g2) + 하단 5행 테이블 (`.tbl-row.cat-for` / `.tbl-row.cat-with` 그룹 컬러)
**핵심 메시지**: "에이전트를 설계할 것인가? 에이전트로 디자인할 것인가?" 두 갈래의 R&R 비교.
**시각 요소**:
- 상단 2카드: Designing FOR Agents (보라톤 마커) / Designing WITH Agents (청록톤 마커)
- 하단 테이블: 5행 (a, b는 FOR / c, d, e는 WITH 카테고리). 좌측 컬러 보더 + 행 배경 틴트로 그룹 시각 구분.

**디자인 의도**
- 비교/대조의 정보 표현이 핵심. 카테고리(FOR vs WITH)가 한눈에 구분되어야 함.
- 테이블 디자인 — 가독성, 행 그룹 컬러 차별화, AS-IS/TO-BE 컬럼 위계
- 카드 ↔ 테이블 컬러 일관성 (FOR=보라 카테고리, WITH=청록 카테고리)

---

### 키스크린 4 — 3.1 Human-in-the-loop (train 플로우 + 카드 패턴)
**위치**: 데크 14번째 / 슬라이드 인덱스 13
**대표 패턴**: 가로 train 플로우 + 3카드 (각 카드 하단 Theory 태그)
**핵심 메시지**: 에이전트가 인간에게 묻는 시점은 정해진 단계가 있고, 그 단계마다 학문적 근거가 있다.
**시각 요소**:
- 상단: 10단계 가로 train (`.train-step` × 10, 일부 하이라이트)
- 하단: 3카드 (Goal Alignment / Decision Support / Adaptive Learning), 각 카드 안에 Theory 태그 (Mental Model / Bounded Rationality 등)

**디자인 의도**
- 정보의 두 층위 — 위(전체 흐름) ↔ 아래(특정 단계의 깊이) 시각적 매핑
- train 단계의 하이라이트 강조 디자인 (어떤 단계가 중요한지)
- 카드 안의 본문 ↔ Theory 태그(메타 정보) 위계 구분

---

### 키스크린 5 — Chap A 간지 (챕터 간지 대표)
**위치**: 데크 18번째 / 슬라이드 인덱스 17
**대표 패턴**: 챕터 간지 (`.s-full .g-chap`)
**콘텐츠**:
- 라벨: "Chap A"
- 본제목: "디자이너가 만드는 것이 달라졌다"

**디자인 의도**
- Part 4 안에서 챕터를 구분하는 시각적 호흡 (Part 표지보다는 한 단계 작은 무게)
- 같은 Part 안의 챕터 일관성 (Chap A / B / C 모두 같은 디자인 언어)
- Part 표지와 톤은 같지만 약간 다른 톤(데크 깊이 표현)

---

### 키스크린 6 — 4.1 화면 기획에서 로직 기획으로 (mockup 캡처 슬라이드 패턴) ⭐ 가장 복잡
**위치**: 데크 19번째 / 슬라이드 인덱스 18
**대표 패턴**: 다크 슬라이드 위에 흰 종이(off-white) 캡처 슬라이드가 그림자와 함께 떠 있고, 그 위에 컬러 사각형 오버레이 + 점선 표시선 + 프로스티드 글래스 콜아웃(번호 배지)이 합성된 형태. 4.2와 함께 "디자이너가 자연어 프롬프트로 시스템 프롬프트를 만든다"는 메타 메시지를 보여줌.

**핵심 메시지**: 디자이너는 와이어프레임 대신 시스템 프롬프트를 자연어로 설계하고, Claude가 그것을 실제 결과물로 변환한다. 화면에 보이는 것이 그 결과물.

**시각 요소 (가장 많은 디자인 결정 필요)**
- **다크 슬라이드 배경** + **흰 종이 캡처 슬라이드** 합성
- 좌상단 메타 주석 (회색 작은 글씨)
- 캡처 슬라이드 헤더: "System Prompt — Trend Research 에이전트 만들기"
- 좌측 컬럼: 워크플로우 SVG (4가지 도구 컬러 코딩) + md 파일 링크
- 우측 컬럼: ① 역할 + ② 단계별 스킬/도구 매칭 테이블 (컬러 범례) + ③ 핵심 산출물
- **3개 오버레이** (반투명 컬러 사각형, 각 섹션을 강조)
- **3개 점선 표시선** (오버레이와 콜아웃 연결)
- **3개 콜아웃** (반투명 흰색 + 컬러 테두리 + 백드롭 블러 + 번호 배지 1/2/3, 자연어 프롬프트 텍스트)

**디자인 의도**
- 메타 레이어(다크 페이지 = 강의 자료) ↔ 콘텐츠 레이어(흰 종이 = 디자이너의 산출물) ↔ 어노테이션 레이어(콜아웃·오버레이 = 강의자의 해설)의 3중 시각 위계가 명확해야 함.
- 콜아웃은 흰 종이의 콘텐츠를 가리지 않으면서도 분명히 보여야 함. 백드롭 블러 + 컬러 테두리가 핵심.
- "디자이너가 자연어로 만든 결과물이 이거다"라는 시연 효과 — 콜아웃의 프롬프트 텍스트를 읽을수록 흰 종이의 결과물이 어떻게 만들어졌는지 이해되어야 함.
- 컬러 코딩(MCP=보라/API=녹/Library=주황/Claude=흰) 의미 보존.

### 키스크린 7 — 4.6 코딩 에이전트와 퀵 프로토타이핑 (proto-grid + 프롬프트 패턴) ⭐ 고유 패턴
**위치**: 데크 24번째 / 슬라이드 인덱스 23
**대표 패턴**: 4컬럼 균등 분할 (L1~L4 레벨), 각 컬럼 세로 흐름 (목적 → 결과물 → 헤더 → 박스 흐름도 → 프롬프트 박스). 박스 흐름은 ↓ 메인, ⋮ 대안 점선.

**핵심 메시지**: 코딩 모르는 디자이너도 4단계 레벨로 점진적으로 작동하는 프로토타입을 만들 수 있다.

**시각 요소**
- 상단: Before / After 2카드
- 하단: **proto-grid** (4컬럼)
  - 각 레벨: 목적 라벨/텍스트 + 결과물 라벨/텍스트 + 레벨 헤더(L1 + 인터랙티브 목업) + 도구 박스 세로 흐름 + 프롬프트 박스
  - 메인 박스(작동) vs 대안 박스(점선 테두리) 시각 구분
  - 프롬프트 박스: 보라톤 좌측 마커 + "PROMPT" 라벨 + 자연어 프롬프트 텍스트
- L4에 작은 주석 (Lovable 대체 가능)

**디자인 의도**
- 4단계의 점진성을 시각적으로 표현 (L1 < L2 < L3 < L4의 복잡도)
- 컬럼 안의 정보 위계 (목적·결과물 메타 → 헤더 → 도구 박스 → 프롬프트)
- 프롬프트 박스가 디자이너의 실용 자산임을 강조 (그대로 복사해서 쓸 수 있는 자료)
- 박스 + 화살표 + 프롬프트 박스의 3가지 다른 시각 언어 통일

---

### 키스크린 8 — Part 4 Conclusion (4-card 결론 패턴)
**위치**: 데크 30번째 / 슬라이드 인덱스 29
**대표 패턴**: 4컬럼 g4, 각 카드 = 한글 제목 + 영문 부제(이탤릭) + 태그라인(굵은 단언) + 불릿 2개. 보라톤 액센트 (불릿 도트).

**핵심 메시지**: 에이전트가 반복을 떠맡은 시간을 디자이너는 어디에 쓸 것인가. 4가지 새로운 격차(Gap).

**시각 요소**
- 카드 4개: 현장 탐구(Physical Grounding) / 지적 심화(Tech Deep-Dive) / 교차 지능(Cross-Sync) / 자산화(Knowledge Assetization)
- 각 카드 구조: 한글 제목 → 영문 부제(이탤릭, 옅음, 하단 보더) → 태그라인 → 불릿 2개

**디자인 의도**
- Part 4 마무리 슬라이드. 진지하고 핵심적인 메시지 전달.
- 4개 카드는 동등한 위계(병렬). 4가지 격차의 무게감이 같음을 표현.
- 한글 제목 → 영문 부제의 위계 (한국어 우선이지만 영문 키워드도 강조 — 이중 언어 표기 룰).
- 태그라인은 짧고 단언적인 핵심 메시지 (각 카드의 "왜").
- 불릿은 보조 설명 (구체 사례/정의).

---

### 키스크린 9 — 5.5 멀티 디바이스 오케스트레이션 (timeline + R&R 섹션 패턴) ⭐ 고유 패턴
**위치**: 데크 36번째 / 슬라이드 인덱스 35
**대표 패턴**: 상단 가로 timeline (워치 → 폰 → 노트북 → TV) + 하단 디자이너 R&R 섹션 (3개 원형 번호 카드, 보라 상하 보더 타이틀)

**핵심 메시지**: 에이전트가 디바이스 간 경계 없이 사용자를 따라다니는 연속 지능. 디자이너는 그 Handoff를 설계한다.

**시각 요소**
- 상단 timeline: 4개 디바이스 박스 + 화살표. 각 박스 = 디바이스 이름(보라톤 강조) + 짧은 설명.
- 하단 R&R 섹션: "디자이너 R&R — 기기 간 경계 없는 지능의 Handoff 설계" 타이틀 (보라 상하 보더, 가운데 정렬, 대문자 letter-spacing) + 3개 원형 번호 카드

**디자인 의도**
- timeline은 시간/공간/디바이스의 흐름을 한 줄로 표현 (← 5.5 고유)
- R&R 섹션은 Part 5 5장 모두에 반복되는 패턴 — 이 시안 결정이 5.1~5.5에 일괄 적용됨
- R&R 카드의 원형 번호 배지가 보라톤 액센트의 일관 사용 — 다른 보라 액센트(콜아웃, 메타 태그 등)와 통일성
- timeline ↔ R&R 사이의 시각적 분리 (다른 정보 층위)

## 7. 디자인 시 고려사항

### 일관성 우선 결정 항목
1. **컬러 팔레트** — 베이스(다크/라이트), 액센트, 카테고리 컬러(MCP/API/라이브러리/Claude 의미 보존)
2. **타이포 시스템** — 헤딩 스케일, 본문 사이즈, 라벨/메타 사이즈
3. **간격 시스템** — 카드/섹션 간 패딩, 컴포넌트 내부 간격
4. **컴포넌트 키트** — 카드, 테이블, 콜아웃, 번호 배지, 화살표/표시선, 오버레이 박스의 통일된 룩

### 시각적 다양성이 필요한 부분
- 9 키스크린은 매우 다른 패턴 — 카드 그리드, 테이블, 가로 train, 세로 proto-grid, mockup 캡처, timeline, R&R 섹션
- 각 패턴의 고유성을 살리되 같은 디자인 시스템으로 묶이는 균형 필요

### 데이/다크 모드 고려
- 현재 다크 베이스. 흰 종이 mockup(4.1)이 다크 위에 떠 있는 합성.
- Claude Design 시안이 데이모드(흰 베이스)로 가더라도 같은 정보 위계가 유지되도록 컴포넌트 설계.

### 정보 밀도
- 슬라이드 강의 자료라 텍스트가 많음. 가독성과 미감의 균형.
- 1920×1080 고정 캔버스에서 글자가 작아도 읽혀야 함 (실제 발표 시 화면 멀리서 봐도).
