# Part 5. 전략과 침투: 에이전트 생태계와 비즈니스 확장

## 5.1 버티컬 에이전트 Preset 전략

**도입 카피:** "특정 목적에 최적화된 에이전트를 단말에 미리 탑재해 Zero-effort 경험 제공"

**페이지 구성:** 상단 비전 시나리오 + 하단 디자이너 R&R 원 3개

### 상단: 버티컬 에이전트 비전 시나리오 (예시)

- **헬스 에이전트** — 생체 센서 데이터를 실시간으로 읽고, 수면·운동·식단을 종합해서 맞춤 루틴을 자율적으로 조정
- **이동 에이전트** — 일정·날씨·교통을 종합해서 출발 시간, 경로, 이동 수단까지 자율 제안·예약
- **커머스 에이전트** — 사용 패턴을 보고 소모품 재주문, 가격 비교, 구매까지 자율 완결

### 하단: 디자이너 R&R (원 3개)
**R&R 타이틀:** "총체적 페르소나와 목적지 설계"

**① 페르소나 정의**
특정 도메인에 최적화된 에이전트의 성격, 말투, 반응 속도 설계

**② 엔드투엔드 시나리오**
목적에 도달할 수 있는 자동화된 여정 기획

**③ 액추에이터 매핑**
하드웨어 피드백과 디지털 서비스를 통합한 다중 출력 경로 설계

## 5.2 특화 지능: 인지와 기억 에이전트

**도입 카피:** "제조사의 강점을 살린 전문 에이전트 분리 배치, Perception Agent와 Memory Agent"

**페이지 구성:** 상단 비전 시나리오 2컬럼 + 하단 디자이너 R&R 원 3개

### 상단: 비전 시나리오 (2컬럼)

**Perception 에이전트가 하는 일 (예시)**
- **감정 인지** — 카메라·마이크·생체 센서로 감지된 신호를 스트레스·집중도·감정 상태의 의미로 추론
- **공간 인지** — 사용자가 있는 공간(회의실, 주방, 침실)의 맥락을 파악해서 상황 정보로 변환

**Memory 에이전트가 하는 일 (예시)**
- **라이프로그** — 하루 동안의 활동, 대화, 이동, 건강 데이터에서 패턴과 인사이트를 추출
- **관계 기억** — 가족·친구와의 대화 패턴, 중요한 날짜, 선호도를 바탕으로 관계 맥락 구성

### 하단: 디자이너 R&R (원 3개)
**R&R 타이틀:** "데이터의 의미 추출과 윤리적 가이드라인 정의"

**① 상황 인지 추상화**
센서에 유입되는 로우 데이터를 어떤 수준의 정보로 변환해 보고할지 인지의 깊이와 맥락적 의미 결정

**② 프라이버시 및 투명성 설계**
인지 에이전트가 백그라운드 상시 작동 시 사용자가 '감시'가 아닌 '보살핌'으로 인지할 수 있도록 돕는 가시적 피드백과 데이터 통제권 설계

**③ 메모리 큐레이션**
방대한 라이프 로그 중 유의미한 가치를 주는 하이라이트 선별, Review Experience 디자인 → 추론의 정합성 확인, 정보의 유의미함과 가치 확인

## 5.3 기술 자산과 네이티브 서비스의 스킬화 Skill-as-a-Service

**도입 카피:** "제조사만의 독보적인 네이티브 서비스를 표준화된 '스킬'로 만들어 배포하고 사용자의 의도에 맞는 스킬을 매칭해주는 스킬 스토어 구축"

**페이지 구성:** 상단 실사례 → 중단 비전 → 하단 디자이너 R&R 원 3개

### 상단: 실사례 레퍼런스
서비스 자산을 스킬로 유통하는 트렌드가 이미 시작됨.
- Stripe, Figma, Atlassian 등이 자사 서비스를 Claude 스킬 생태계에 공식 참여
- Google Stitch의 DESIGN.md, brand.md 등 브랜드·디자인 가이드를 AI가 읽을 수 있는 md 형식으로 공개하는 오픈 스탠다드 등장
- Anthropic, Vercel, Cloudflare, Stripe 등 주요 기업들이 자사 기술 문서를 llms.txt 형식으로 공개

### 중단: 우리가 할 수 있는 것 (비전)
단말 제조사 가상 시나리오:
- 카메라 야간 촬영 알고리즘을 스킬로 패키징 → 외부 에이전트가 "사진 찍어줘"라고 하면 자동 호출
- 노이즈 캔슬링 기술을 스킬로 배포 → 써드파티 AI 어시스턴트가 상황에 맞게 자율 활용
- 센서 데이터 분석 기술을 스킬로 제공 → 헬스·이동·커머스 에이전트가 필요시 호출

### 하단: 디자이너 R&R (원 3개)
**R&R 타이틀:** "기술의 규격화와 브랜드 어트리뷰션"

**① 인텐트 스키마 설계**
우리 하드웨어의 기능을 외부 AI가 오해 없이 호출할 수 있도록 기능의 이름과 범위 규격화

**② 브랜드 아이덴티티 유지**
타사 에이전트가 우리 브랜드의 기술력임을 사용자가 인지할 수 있는 시청각 단서 설계

**③ 권한 승인 UX**
외부 지능이 우리 센서에 접근할 때, 사용자가 불안하지 않게 제어권을 주는 승인 인터페이스

## 5.4 No-code 에이전트 조립 도구

**도입 카피:** "사용자가 코딩 없이 본인의 필요에 맞는 에이전트를 직접 조립하거나 커스텀할 수 있는 단말 전용 개발 환경 제공"

**페이지 구성:** 상단 비전 시나리오 + 하단 디자이너 R&R 원 3개

### 상단: 비전 시나리오 (예시)

- **모바일 에이전트 스튜디오** — OpenClaw처럼 스킬 단위로 에이전트를 조립하는 개념을 터치 기반 UI로 구현. 사용자가 블록을 드래그해서 "아침 7시에 날씨 확인 → 일정 체크 → 알람 조정" 같은 루틴을 직접 만드는 환경
- **단말 제조사 스킬 마켓** — 필요한 스킬을 골라서 자기 에이전트에 붙이는 방식. 스마트폰 앱을 설치하듯 에이전트 능력을 확장

### 하단: 디자이너 R&R (원 3개)
**R&R 타이틀:** "복잡한 논리의 시각적 도구화"

**① 노코드 인터페이스 설계**
복잡한 로직을 자연어 또는 레고 블록처럼 쉽게 조립할 수 있는 툴킷 디자인

**② AI 역량 가시화**
에이전트가 할 수 있는 일과 없는 일을 직관적으로 이해할 수 있게 돕는 메타포 설계

**③ 디버깅 UX**
사용자가 만든 에이전트가 예상과 다르게 동작할 때, 그 이유를 쉽게 파악하고 수정할 수 있는 피드백 화면 설계

## 5.5 멀티 디바이스 오케스트레이션

**도입 카피:** "폰, 워치, 가전 등 제조사 생태계 내에서 에이전트가 중단 없이 사용자를 따라다니며, 상황에 가장 적합한 기기가 주도권을 이어받는 연속적 지능 환경 구축"

**페이지 구성:** 상단 연속 시나리오 타임라인 + 하단 디자이너 R&R 원 3개

### 상단: 연속 시나리오 타임라인
워치 → 폰 → 노트북 → TV 순서로 에이전트가 주도권을 이어받는 하루 흐름

- **워치** — 아침 루틴 에이전트가 기상·건강 체크로 하루 시작
- **폰** — 집을 나서면 교통·일정 안내로 이어받음
- **노트북** — 회사 도착 후 오늘 브리핑·태스크 정리로 전환
- **TV** — 귀가 후 하루 요약·다음 일정 안내로 마무리

### 하단: 디자이너 R&R (원 3개)
**R&R 타이틀:** "기기 간 경계 없는 지능의 Handoff 설계"

**① 컨텍스트 연속성 설계**
기기가 바뀌어도 에이전트가 대화·상태·진행 중인 태스크를 그대로 이어받을 수 있도록 정보의 흐름과 인계 시점을 정의

**② 주도권 전환 신호 설계**
어떤 상황에서 어떤 기기가 주도권을 가져야 하는지 판단 기준과 전환 순간을 사용자가 자연스럽게 인지할 수 있는 시각·청각·촉각 피드백 설계

**③ 멀티 디바이스 UI 문법 정의**
폰·워치·가전 등 각기 다른 화면 크기와 인터페이스 조건에서 에이전트가 일관된 경험을 제공할 수 있도록 기기별 표현 규칙과 공통 언어 정립
